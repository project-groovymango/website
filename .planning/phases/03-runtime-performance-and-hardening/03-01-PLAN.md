<task type="auto">
  <name>Task 2: Add Content Security Policy meta tag</name>
  <files>index.html</files>
  <action>
Add a CSP meta tag to `index.html` in the `<head>` section, after the resource hints block and before the Basic SEO block.

The entire content attribute MUST be on a single line (newlines break CSP parsing):

```html
<meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self' 'unsafe-inline' https://app.cal.com; style-src 'self' 'unsafe-inline'; img-src 'self' data:; frame-src https://app.cal.com https://*.cal.com; connect-src 'self' https://app.cal.com https://*.cal.com; font-src 'self';">
```

Directive rationale:
- `script-src 'unsafe-inline'`: Required for Vite's module loading script tag (`<script type="module" src="/src/main.jsx">` in dev mode). Can be tested and potentially removed for production builds later.
- `style-src 'unsafe-inline'`: Required because React inline `style={{...}}` props inject element styles.
- `img-src data:`: Required because Vite may inline small SVGs as data URIs.
- `frame-src *.cal.com`: Cal.com embed loads an iframe from cal.com subdomains.
- `connect-src *.cal.com`: Cal.com embed makes API calls to cal.com.

Also remove the existing `<link rel="preconnect" href="https://cal.com" crossorigin />` -- since Cal.com is now deferred to click, preconnecting on page load is wasteful. The preconnect was added in Phase 1 when Cal.com loaded eagerly.
