<task type="auto">
  <name>Task 2: Add explicit width and height to all img elements</name>
  <files>src/index.jsx, src/components/ToolsMarquee.jsx</files>
  <action>
Add explicit `width` and `height` HTML attributes to every `<img>` element in the codebase. This is needed even when inline styles set dimensions, because HTML attributes reserve layout space before CSS loads (preventing CLS).

**src/index.jsx changes:**

1. Hero frog logo (line ~55): Already has `style={{ width: "48px", height: "48px" }}`. Add HTML attributes:
   `<img src={frogWebp} alt="a2labs logo" width={48} height={48} style={{ width: "48px", height: "48px", position: "relative", top: "-6px" }} />`

2. Logo grid images (line ~85): Currently `<img src={src} alt={alt} height={24} />`. Need to add width. SVG logos at height=24 have varying widths -- measure each:
   - Open each SVG file in `src/assets/` and check the viewBox to compute width at height=24
   - Add a `width` property to each entry in the LOGOS array in `src/data/content.js`, OR add a default width attribute to the img tag
   - Practical approach: Since these are SVGs that scale proportionally and all have `height={24}`, add `width="auto"` -- but this does NOT prevent CLS. Better: add a fixed reasonable width estimate. Since SVG logos typically have aspect ratios around 3:1 to 5:1 at height=24, measure each SVG's viewBox:
     - Read each SVG file's viewBox attribute
     - Compute: width_at_24 = (viewBox_width / viewBox_height) * 24
     - Add the computed width to each LOGOS entry in content.js

3. Case study logos (lines ~102, ~113): Currently `<img src={logo} alt={logoAlt} height={14} />`. Same approach: compute width from SVG viewBox at height=14 and add to CASE_STUDIES entries in content.js, then render: `<img src={logo} alt={logoAlt} width={cs.logoWidth} height={14} />`

4. Team photo (line ~207): `<img className="team-img" src={aminPhoto} alt="Amin Laanaya" loading="lazy" />`. Measure actual dimensions of amin.webp (use Node: `identify` or read the file metadata). Add width and height attributes matching the natural image dimensions (or the display dimensions if CSS constrains them).

5. n8n logo in FAQ (line ~229): `<img src={n8nLogo} alt="n8n" style={{ height: 16 }} />`. Read n8n-logo.svg viewBox, compute width at height=16, add: `<img src={n8nLogo} alt="n8n" width={computed} height={16} style={{ height: 16 }} />`

6. Footer frog (line ~257): Already has `style={{ width: "96px", height: "96px" }}`. Add HTML attributes:
   `<img src={frogWebp} alt="a2labs logo" width={96} height={96} style={{ width: "96px", height: "96px" }} />`

**src/components/ToolsMarquee.jsx changes:**

7. Marquee logos: Currently render with `loading="lazy" decoding="async"` and optional `style={height ? { height } : undefined}`. Need width and height on each:
   - Add `width` and `height` properties to each entry in the ToolsMarquee LOGOS array (measure SVG viewBoxes)
   - Default height for most is 16px (the marquee item CSS likely constrains this -- check the CSS)
   - Render: `<img ... width={width || defaultWidth} height={height || 16} />`
   - For the MakeLogo component entry, no change needed (it's an inline SVG, not an img)

**src/data/content.js changes:**

8. Add `width` field to LOGOS and CASE_STUDIES entries so the img elements can render with both width and height.

To measure SVG dimensions: For each SVG file, read the file, parse the viewBox attribute (`viewBox="0 0 W H"`), and compute the display width as `Math.round((W / H) * displayHeight)`.
